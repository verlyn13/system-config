# GitHub MCP Integration

Single source of truth for the GitHub MCP integration on this system.

For the general MCP framework (scopes, launch patterns, sync behavior),
see [`docs/mcp-config.md`](./mcp-config.md). For 1Password policy, see
[`docs/secrets.md`](./secrets.md).

This doc replaces the prior migration plan once the work it tracked
landed (commits `a8e0b64` through `fcf59fc`).

## Architecture

| Host | Auth | Config shape | Wrapper? |
|---|---|---|---|
| Claude Code CLI | PAT via stdio wrapper | stdio entry pointing at `~/.local/bin/mcp-github-server` | **yes** |
| Cursor | PAT via stdio wrapper | same stdio entry | **yes** |
| Codex CLI | PAT via `bearer_token_env_var = "GITHUB_PAT"` | native remote (TOML) | no |
| Windsurf | OAuth 2.1 + PKCE (native, shipped 1.12.41) | `serverUrl` only | no (tentative — may need wrapper fallback) |
| Copilot CLI | Copilot-internal | **no synced entry** (built-in `github-mcp-server`) | n/a |

**Why Claude Code uses the wrapper:** Claude Code's MCP SDK performs OAuth
discovery (RFC 9728 Protected Resource Metadata) before honoring static
Authorization headers. GitHub's remote MCP advertises OAuth pointing at
`github.com/login/oauth`, which does not support Dynamic Client
Registration. Claude Code's SDK errors out with "SDK auth failed:
Incompatible auth server: does not support dynamic client registration"
before falling back to our static bearer. The stdio wrapper sidesteps
this — Claude Code sees a plain stdio server, no OAuth discovery layer
to follow, and `mcp-remote` handles the HTTPS call with a pre-
authenticated bearer. Cursor has the same SDK behavior; same fix.

Rendered shapes (live, verified):

```json
// ~/.claude.json  (Claude Code)
"github": { "type": "stdio", "command": "/Users/verlyn13/.local/bin/mcp-github-server" }

// ~/.cursor/mcp.json  (Cursor)
"github": { "type": "stdio", "command": "/Users/verlyn13/.local/bin/mcp-github-server" }

// ~/.codeium/windsurf/mcp_config.json  (Windsurf)
"github": { "serverUrl": "https://api.githubcopilot.com/mcp/" }
```

```toml
# ~/.codex/config.toml  (Codex)
[mcp_servers.github]
url = "https://api.githubcopilot.com/mcp/"
bearer_token_env_var = "GITHUB_PAT"
http_headers = { "X-MCP-Toolsets" = "context,repos,issues,pull_requests,users,actions,code_security,dependabot,discussions,gists,labels,orgs,projects,releases,security_advisories,secret_protection,github_support_docs_search" }
```

All shapes are generated by `scripts/sync-mcp.sh`. Do not hand-edit these
entries; rerun sync instead. Hand-added sub-tables like
`[mcp_servers.github.tools.create_pull_request]` are supported and
survive regeneration if placed **outside** the `# BEGIN … # END` markers
in the Codex TOML.

## Secret (1Password)

| Property | Value |
|---|---|
| Account | `my.1password.com` |
| Vault | `Dev` |
| Item | `github-mcp` |
| Field | `token` |
| op URI | `op://Dev/github-mcp/token` |
| Env var name | `GITHUB_PAT` |
| Rotation | manual, fine-grained PAT with fixed expiry (90 days or 1 year) |

The 1P item is dedicated to the MCP integration. Do not reuse
`op://Dev/github-dev-tools/token` (that entry serves `gh` CLI and other
general dev tooling).

Rotation procedure:
1. GitHub → Settings → Developer settings → Personal access tokens →
   Fine-grained tokens → `github-mcp` → Regenerate.
2. Update the `token` field on the `github-mcp` item in 1Password.
3. Verify: `op read --account my.1password.com "op://Dev/github-mcp/token" >/dev/null && echo ok`
4. If the rotation changes toolset scopes, update the PAT scope set
   below and the `X-MCP-Toolsets` header in `scripts/sync-mcp.sh` in
   lockstep, then rerun sync.

## Fine-grained PAT scopes

Target repository permissions (for the repos the MCP should see):

| Permission | Level |
|---|---|
| Metadata | Read |
| Contents | Read & write |
| Issues | Read & write |
| Pull requests | Read & write |
| Actions | Read & write |
| Workflows | Read & write |
| Commit statuses | Read |
| Deployments | Read |
| Discussions | Read & write |
| Code scanning alerts | Read & write |
| Secret scanning alerts | Read |
| Dependabot alerts | Read |
| Repository security advisories | Read & write |
| Administration | Read |

Account permissions: Profile (Read), Gists (Read & write), Starring
(Read).

Organization permissions: Metadata (Read), Members (Read), Administration
(Read), Custom properties (Read), Projects (Read & write).

**Not granted:** Secrets, Dependabot secrets, Environments, Pages,
Variables, Webhooks, Single file, Notifications.

**Scope filtering note:** fine-grained PATs do **not** get server-side
tool filtering (classic PATs do, as of Jan 28, 2026). The full GitHub
MCP tool surface would be exposed by default. The curated
`X-MCP-Toolsets` header below is the mechanism that enforces our
intended scope.

## Curated toolsets

URL: `https://api.githubcopilot.com/mcp/` (base, no `/x/…` suffix).

`X-MCP-Toolsets` header (17 toolsets, aligned with PAT scopes above):

```
context,repos,issues,pull_requests,users,actions,code_security,dependabot,discussions,gists,labels,orgs,projects,releases,security_advisories,secret_protection,github_support_docs_search
```

Dropped (intentionally): `notifications` (no account-notifications PAT
permission), `stargazers` (low-value noise), `git` (redundant with
`contents` tools), `copilot` + `copilot_spaces` (not licensed here),
`experiments` (unclear scope).

Approximate tool count: 68 (down from `/x/all`'s 89).

The list lives in two places that must stay in lockstep:

1. `scripts/sync-mcp.sh` — `GITHUB_TOOLSETS` constant (used for Claude
   Code, Codex, and the wrapper via the same script)
2. `home/dot_local/bin/executable_mcp-github-server.tmpl` —
   `TOOLSETS` constant

Any edit to either must also update the other and rerun `chezmoi apply`
+ `scripts/sync-mcp.sh`.

## Launch patterns

### Codex CLI (needs `${GITHUB_PAT}` in process env)

```bash
op run --account my.1password.com \
       --env-file=$HOME/.config/mcp/common.env -- codex
```

Bare `codex` still starts, but GitHub MCP returns 401 on every call
until relaunched under `op run`. `common.env` is the shared manifest
for all `op://`-backed MCP secrets; see
[`docs/mcp-config.md`](./mcp-config.md#secret-handling).

### Claude Code CLI and Cursor (stdio wrapper)

Launch normally. The stdio wrapper at `~/.local/bin/mcp-github-server`
reads `op://Dev/github-mcp/token` at subprocess start — works regardless
of how the host was launched. Launching via `op run --env-file=…` is
still fine and makes the wrapper fast-path (skips its own `op read`
when `GITHUB_PAT` is already in the inherited env), but it is not
required.

### Windsurf (GUI)

Launch normally. On first GitHub MCP interaction, Windsurf initiates
OAuth 2.1 + PKCE via GitHub's Protected Resource Metadata discovery
(RFC 9728). Consent once; Windsurf manages refresh tokens.

**Fallback:** if OAuth fails, the Windsurf version is below 1.12.41, or
refresh tokens stop working, revert Windsurf to the stdio wrapper by
changing `GITHUB_WINDSURF` in `scripts/sync-mcp.sh` to the same shape as
`GITHUB_CURSOR` and rerunning sync. One-commit revert.

### Copilot CLI

No configuration. The built-in `github-mcp-server` ships with the tool
and uses your Copilot credentials. Customize per-session:

```bash
copilot --enable-all-github-mcp-tools           # all toolsets
copilot --add-github-mcp-toolset <name>         # add a specific toolset
copilot --disable-builtin-mcps                  # turn it off
/mcp show github-mcp-server                     # from inside a session
```

## Troubleshooting

| Symptom | Likely cause | Fix |
|---|---|---|
| Claude Code / Cursor github status: failed, `SDK auth failed: Incompatible auth server: does not support dynamic client registration` | Config still uses native `type: "http"`; SDK is attempting OAuth DCR against GitHub | Rerun `scripts/sync-mcp.sh` — Claude Code and Cursor must use the stdio wrapper entry, not native HTTP. |
| Codex github MCP: 401 | Codex launched without `op run`; `GITHUB_PAT` missing from process env | Relaunch `codex` via `op run --env-file=~/.config/mcp/common.env -- codex` |
| Wrapper (Claude/Cursor): "set GITHUB_PAT or store PAT in op://Dev/github-mcp/token" | `op` can't reach the item; 1Password desktop integration off | Enable 1P Settings → Developer → Integrate with 1Password CLI; confirm `op vault get Dev --account my.1password.com` |
| Windsurf: stuck on connecting / no OAuth prompt | Windsurf version below 1.12.41 or OAuth discovery blocked | Upgrade Windsurf; if persistent, revert to stdio wrapper (see Fallback above) |
| Tool 403s at call time (e.g. `actions_run_trigger`) | PAT scope missing for that toolset | Add the missing Repository/Organization permission to the PAT; rotate via procedure above |
| Duplicate `[mcp_servers.*]` in Codex TOML | Sync ran against a Codex config without BEGIN/END markers | Wrap the existing `[mcp_servers.*]` block with `# BEGIN system-config managed MCP servers` / `# END …`, then rerun sync |
| Hand-added `[mcp_servers.github.tools.*]` disappeared after sync | Sub-table was inside the managed BEGIN/END block | Move the sub-table below the `# END` marker; TOML merges it into the parent table regardless of position |

## Rollback

Each architectural commit is independently reversible:

- Revert `fcf59fc` (per-host split) → returns to uniform wrapper across all three stdio hosts plus Codex bearer.
- Revert `754e45f` (wrapper uses mcp-remote) → returns to the prior stdio server; not recommended (deprecated npm package).
- Revert the entire stack → returns to pre-migration state.

After any revert, rerun `scripts/sync-mcp.sh` to propagate the rolled-back
shapes to live configs.
